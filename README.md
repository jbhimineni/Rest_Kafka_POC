# REST Kafka POCA Spring Boot proof-of-concept application that integrates REST, Kafka, and an H2 in-memory database to handle employee data.  Whenever an employee record is posted via the REST API, it is published to Kafka (running via Docker). A Kafka listener consumes the message and stores it in the database.## 🚀 Features- REST API endpoints to create and fetch employee records- Kafka producer to send employee data- Kafka consumer to listen and persist data- H2 in-memory database for persistence- Spring Data JPA for ORM- Gson for JSON processing- Docker Compose for Kafka setup## 🛠 Technologies Used- Java 21- Spring Boot- Spring Data JPA- Apache Kafka- H2 Database- Gson- Lombok- Docker & Docker Compose- IntelliJ IDEA### 🔍 Verify Your Environment```java -versionmvn -vdocker --versiondocker-compose --version```## Getting Started- Clone from the repository     `git clone https://github.com/jbhimineni/Rest_Kafka_POC.gitcd Rest_Kafka_POC`- Build the app in local   - `mvn install`- Start the SpringBoot App  - `mvn spring-boot:run`- Check the Docker is Health status- Start using the Application  - `mvn spring-boot:run -Dspring-boot.run.arguments="--spring.profiles.active=dev" -DskipTests`### Explanation:1. **Prerequisites**    -
