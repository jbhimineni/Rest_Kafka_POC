# REST Kafka POCA Spring Boot proof-of-concept application that integrates REST, Kafka, and an H2 in-memory database to handle employee data.  Whenever an employee record is posted via the REST API, it is published to Kafka (running via Docker). A Kafka listener consumes the message and stores it in the database.## Features- REST API endpoints to create and fetch employee records- Kafka producer to send employee data- Kafka consumer to listen and persist data- H2 in-memory database for persistence- Spring Data JPA for ORM- Gson for JSON processing- Docker Compose for Kafka setup## Technologies Used- Java 21- Spring Boot- Spring Data JPA- Apache Kafka- H2 Database- Gson- Lombok- Docker & Docker Compose- IntelliJ IDEA### Prerequisites- Java 21+- Maven- Docker & Docker Compose### 🔍 Verify Your Environment```java -versionmvn -vdocker --versiondocker-compose --version```## 🚀 Getting Started- Clone from the repository     - `git clone https://github.com/jbhimineni/Rest_Kafka_POC.git    - `cd Rest_Kafka_POC`- Build the app in local   - mvn install- Start the SpringBoot App  - mvn spring-boot:run- Check the Docker is Health status- Start using the Application  - `mvn spring-boot:run -Dspring-boot.run.arguments="--spring.profiles.active=dev" -DskipTests`### 🛠 Explanation:1. **Prerequisites**    - Explained below required technologies `Prerequisites`2. **Setup Instructions**    - Set the Java path, Install the Docker(Test the docker trial run)   - Kafka should be able to run in the docker   - Spring Boot<V>, Maven should compact with Java <21>   - Set up the DataBase(Here H2 Will be automatically added while app Booting time)   3. **Testing** When running the application in local     - Check the server logs on console     - When application is started successfully, hit the below request from terminal   ```bash       curl -X POST http://localhost:8081/employee/create \ -H "Content-Type: application/json" \-d '{"firstName":"Jagades","lastName":"Bhimineni","location":"Frisco","age":12,"salary":0.0}'   ```   -After posted the record it will be stored into the DB4. **Access H2 Console**   - Access the DB from the browser while working the SpringApp   - http://localhost:8081/h2-console/login.jsp   - login with the User `rest_kafka_poc` Password `rest_kafka_poc`   5. **API Endpoints**     - Since the POC is dealing the `Employee` data the base uri will be `/employee`    - The `/employee/` handel the index page it indicates app is successfully handel the web requests    - The `/employee/create` Will take the `JSON` formate input in employee record and sends to kafka    - The `/employee/get` fetches `id` associated employee record and send the record in response body6. **Troubleshooting**    - While working with the Docker container it will create the kafka in docker     - `compose-docker` Will manages to create the kafka container and making it available for spring app      - If Zookeeper is failed to start or kafka is failed to start can manually make them run is possible     - Container ports are open and could be able to locate on the same ports we should run the processor manually `i.e: pkill -9 <processid(macOS)>`    - If SpringBoot tests are failing not necessary to run it possible to skip also reduce the start time ### Sample run captures- Test Results and CodeCoverage report attached to the repo path under the     ` src/main/resources/runresult`
